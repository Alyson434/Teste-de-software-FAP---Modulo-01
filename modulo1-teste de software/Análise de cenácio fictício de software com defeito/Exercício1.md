# 📋 ***Relatório – Análise de Defeitos do Sistema PsicoCare***

## 1️⃣ Em que fase do ciclo o defeito poderia ter sido encontrado?

Os defeitos poderiam ter sido encontrados principalmente na **fase de testes**, antes da entrega do sistema para os usuários. Nessa etapa, a equipe verifica se todas as funções estão funcionando corretamente e se o sistema atende aos requisitos definidos.

---

## 2️⃣ Impactos do defeito para o usuário e para o negócio

Os defeitos podem causar diversos problemas, como:

- ❌ O cadastro de pacientes pode não ser salvo, causando perda de informações importantes.
- 📅 A agenda pode permitir duas consultas no mesmo horário para o mesmo psicólogo, gerando conflitos de atendimento.
- 📝 Cadastros com informações inválidas podem ser aceitos, diminuindo a confiabilidade do sistema.
- 💰 O controle financeiro pode registrar valores negativos, causando erros nas contas da clínica.
- 📦 O estoque pode ficar com quantidade negativa, dificultando o controle dos materiais.
- 🗑️ A exclusão de pacientes pode deixar consultas sem vínculo, gerando dados inconsistentes.
- 🔍 A pesquisa de pacientes pode dificultar a localização das informações.
- 🔒 O login sem bloqueio após várias tentativas deixa o sistema mais vulnerável a acessos indevidos.

### Impactos gerais

Esses problemas podem causar:

- Perda de dados;
- Insatisfação dos usuários;
- Prejuízo financeiro;
- Retrabalho para os funcionários;
- Diminuição da confiança no sistema.

---

## 3️⃣ Existe relação com o gráfico do custo de correção de bugs?

**Sim.**

O gráfico mostra que, **quanto mais cedo um defeito é encontrado, menor é o custo para corrigi-lo**.

Como esses problemas foram identificados durante a **fase de testes**, a correção será mais simples e mais barata do que se fossem descobertos somente depois que o sistema estivesse sendo utilizado pela clínica.

Se os defeitos chegassem ao ambiente de produção, poderiam causar:

- 💸 Prejuízos financeiros;
- 📂 Perda de informações;
- 😟 Insatisfação dos clientes;
- 🔧 Maior custo para correção.
