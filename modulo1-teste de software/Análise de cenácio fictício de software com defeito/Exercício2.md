# 📊 Quadro Comparativo – Práticas de QA

| **Práticas Tradicionais de QA** | **Práticas Modernas de QA** |
|---------------------------------|-----------------------------|
| Os testes são feitos no final do desenvolvimento. | Os testes começam desde o início do projeto (**Shift Left**). |
| A maioria dos testes são manuais. | Uso de testes automatizados para ganhar velocidade e qualidade. |
| Testadores trabalham separados da equipe de desenvolvimento. | Desenvolvedores, testadores e demais profissionais trabalham juntos (**Agile Testing**). |
| A correção dos erros acontece mais tarde. | Os erros são encontrados e corrigidos o mais cedo possível. |
| A segurança costuma ser verificada apenas no final. | A segurança faz parte de todo o processo (**DevSecTestOps**). |
| As entregas são maiores e menos frequentes. | As entregas são pequenas, rápidas e contínuas. |

---

# 💭 Reflexão

Na minha opinião, um dos maiores desafios atuais de QA é acompanhar a velocidade com que os sistemas são desenvolvidos. As equipes precisam entregar novas funcionalidades rapidamente, mas sem deixar a qualidade de lado.

Outro desafio é criar testes automatizados eficientes e encontrar os defeitos logo no início do desenvolvimento. Isso ajuda a economizar tempo, reduzir custos e evitar problemas para os usuários.

Por isso, as práticas modernas de QA são importantes, pois ajudam a entregar sistemas mais seguros, confiáveis e com melhor qualidade.
