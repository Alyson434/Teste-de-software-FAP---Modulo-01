# 📚 Atividade Avaliativa – Ágil x Cascata

## 📊 Tabela Comparativa

| Critério                        | 🚀 Ágil                                                                                                                                    | 🌊 Cascata                                                                                                                           |
| ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------ |
| **🧪 Como os testes acontecem** | Os testes são realizados durante todo o desenvolvimento, geralmente a cada sprint. Os defeitos são identificados e corrigidos rapidamente. | Os testes são realizados ao final do desenvolvimento, após a conclusão das demais etapas do projeto.                                 |
| **✅ Vantagens**                 | - Entregas frequentes.<br>- Maior flexibilidade para mudanças.<br>- Feedback contínuo do cliente.<br>- Correção rápida de falhas.          | - Processo bem estruturado.<br>- Fácil documentação.<br>- Melhor para requisitos bem definidos.<br>- Maior previsibilidade de prazo. |
| **⚠️ Limitações**               | - Exige comunicação constante.<br>- Dependência da participação do cliente.<br>- Pode dificultar estimativas iniciais.                     | - Pouca flexibilidade para alterações.<br>- Problemas podem ser descobertos tardiamente.<br>- Correções costumam ser mais custosas.  |

---

## 🔍 Resumo

* **🚀 Ágil:** ideal para projetos dinâmicos e que sofrem mudanças frequentes.
* **🌊 Cascata:** recomendado para projetos com requisitos estáveis e bem definidos.

## 📝 Conclusão

O modelo **Ágil** promove testes contínuos e entregas incrementais, contribuindo para uma rápida identificação de problemas e maior adaptação às necessidades do cliente. Já o modelo **Cascata** segue uma abordagem sequencial, sendo mais adequado para projetos com escopo definido desde o início.

Ambas as metodologias possuem vantagens e limitações, e a escolha entre elas dependerá das características e objetivos do projeto.
